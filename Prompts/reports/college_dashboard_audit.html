6 HIGH 🔴 OPEN EntityLogo.js
lib/utils.js		 EntityLogo uses Clearbit logo guessing from website domain.
The previous hard-coded IITM/NIT/Infosys/TechCorp mappings are gone, but the component now falls back to https://logo.clearbit.com/{domain} derived from the entity's website field. This is still guessing — Clearbit is a third-party service that may return wrong logos, fail silently, or return logos for unrelated domains. 		Current chain: logoUrl prop (saved upload) → Clearbit from website domain → initials fallback.

The Clearbit step is not hardcoding per se, but it is still an assumption: any entity whose website domain matches a Clearbit entry gets that logo, regardless of whether it's correct. If a college uses a subdomain or shared hosting, the wrong logo appears.

Action needed: Remove the Clearbit step from getEntityLogoUrl. Use only explicit logoUrl (uploaded/pasted) → initials. Let users configure their own logo via settings. This is the rule: editable and persisted, not guessed.
